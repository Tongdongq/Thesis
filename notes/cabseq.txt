CACHE_ABSEQ distributions

the first (in time) accessed a[y] is does not always have the lowest address (in forward wave), so it should not be used as an anchor to start caching from. Measurements have been done to determine the optimal offset to start caching from.

Column:
1: number of reported distances
2: value of reported distances (could be a range)
3: percentage of value wrt all distances in category
4: name of category, or sum of all previous percentages (including) to indicate 95% point

Name of category consists of:
- aseq or bseq read
- distance from first observed abseq access to highest/lowest address
- forward or reverse wave

Note that reading distance for forward wave is towards 'high', and for reverse wave is towards 'low'.
